Hard Disk Images

Overview:

A hard disk controller (or interface) card will typically be plugged into slot 7 (or slot 5) via the AppleWin Configuration tab labelled Slots.

There is provision to connect 8 hard disks (HDDs) to each card from the UI (or via the command line). This is done by using .hdv or 800KB .2mg files on your PC. Each hard disk can have a maximum capacity of 32MB.

NB. The hard disk controller supports both fixed-disk types (up to 32MB) and 3.5" floppy types (800KB).

On booting, the Apple II will always attempt to autoboot from slot 7 first. If the hard disk controller has no HDD images connected then the Apple II will attempt to boot from slot 6 (with the Disk][ interface card in it).

To boot a floppy disk with the hard disk controller enabled, either hold down the Open-Apple key during an Apple II restart or issue PR#6 from an AppleSoft prompt.

The firmware in the hard disk controller card defaults to:

  • II/II+/Unenhanced //e: ProDOS block mode, supporting up to 4 HDDs.
  • Enhanced //e: SmartPort, supporting up to 8 HDDs.

    • Warnings:

    • Removing the hard disk controller or a hard disk whilst the Apple II is running can lead to a corrupt .hdv image if the Apple II is writing to the hard disk when it is unplugged!
    • Equally, exiting AppleWin can have the same affect.
    • Copy][+ v7.1 locks up when trying to calculate how many free blocks are available when running a catalog. This is a bug in Copy][+ which is fixed in later versions.
